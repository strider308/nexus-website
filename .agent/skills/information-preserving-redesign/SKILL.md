# Information-Preserving Redesign Skill

Use this skill before redesigning, refactoring, or converting any existing Nexus page or section.

Core rule:
Preserve information before changing presentation.

Before changing a page, inventory:

* route
* section IDs
* headings
* paragraphs
* CTAs
* links
* anchors
* proof claims
* disclaimers
* diagrams
* accessibility requirements
* SEO metadata

For every change, define:

* what stays in HTML
* what becomes visually enhanced
* what gets a fallback
* what must never move into canvas
* what can be removed as redundant
* what must be preserved exactly

Never remove:

* legal pages
* sitemap
* robots
* metadata
* canonical URLs
* accessibility helpers
* reduced-motion behavior
* rollback flags
* production CTAs
* proof-system details
* case-study anchors
* resource links

Final check:
The redesigned version must contain the same or better information, not less.
